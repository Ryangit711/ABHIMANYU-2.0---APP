# HARD KERNEL RULE – SYMBIOTIC DNA CADENCE

**Purpose**: Immutable end‑to‑end workflow for every target company from application to first paycheck. Enforced on every pipeline or cadence change.

1. **Scope**: All companies in TICS pipelines. Roles failing hard filters (credit‑check, heavy‑Excel/quant) are excluded before cadence starts.
2. **DNA Layers**:
   - DNA‑1 Live Intelligence (nightly LIFTOFF scrape)
   - DNA‑2 Role‑DNA mapping (ATOMIZE JD)
   - DNA‑3 Cadence‑DNA (Leg 0‑5 autogenerated in CADENCE_FOOTER.md)
   - DNA‑4 Evaluation‑DNA (EVAL must PASS ≥70 before any DOCX is saved)
   - DNA‑5 Offer‑DNA (salary/eq benchmarks from 16_COMPENSATION_DB.md)
   - DNA‑6 Payroll‑DNA (bi‑weekly schedule, reminder in payroll_reminder.md)
3. **Cadence** (Leg 0 → Paycheck):
   - Leg 0 Referral/Endorsement → `CADENCE UPDATE Leg 1`
   - Leg 1 Application submission → PIPELINE status ✅ SUBMITTED
   - Leg 2 Hiring‑manager name → `CADENCE UPDATE Leg 2` (LinkedIn connect)
   - Leg 3 LinkedIn connection → `CADENCE UPDATE Leg 3`
   - Leg 4 Value‑add email → `CADENCE UPDATE Leg 4`
   - Leg 5 Final nudge → `CADENCE UPDATE Leg 5`
   - Phone‑screen, case study, panel interview, offer, negotiation, onboarding, first paycheck – each step updates PIPELINE and logs in thought journal.
4. **Hard Enforcements**:
   - EVAL WARN/FAIL → pipeline set to ❌ BLOCKED, automatic resume rewrite required.
   - Dual‑write: every DOCX/PDF must exist in Linux worktree AND OneDrive; missing aborts.
   - Every state change must append a line to `data/thought_log/YYYY‑MM‑DD.md`.
   - After each state‑changing command the system runs `git add -A && git commit -m "[action] — [company] — <timestamp>" && git push`. Failure aborts and raises error.
   - `CADENCE_FOOTER.md` is appended to every system response unconditionally.
5. **Example for Seaspan** (already in pipeline): after Kash provides hiring‑manager name, the system will auto‑advance Leg 3‑5, schedule phone‑screen, run `INTERVIEW‑PREP`, handle offer via `NEGOTIATE`, and finally log first paycheck.

*This rule is immutable and governs all future job‑application cadences.*
